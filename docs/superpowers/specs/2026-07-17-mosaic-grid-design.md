# Mosaic Grid 优化设计

## 目标

将第四个效果 Mosaic Grid 从当前的固定散落拼贴，优化为更有秩序、画布利用率更高的非对称网格；点击单张卡片后聚焦放大查看，退出后恢复原位。

## 交互设计

- 初始状态保留 7 张卡片，但使用稳定的 4 列非对称布局。
- 卡片之间使用统一间距，布局整体扩大到舞台的主要可视区域，减少四周空白。
- 点击卡片时，该卡片平滑移动到舞台中心并放大；其他卡片保持原有位置，但降低存在感。
- 聚焦状态下，滚轮只调整当前卡片的缩放，不触发页面滚动。
- 再次点击当前卡片、按 `Escape` 或点击舞台空白区域时，卡片恢复到原始位置和缩放。
- 卡片仍使用真实图片或 Slot 占位内容，保留当前软件的边框、字体和颜色体系。

## 响应式布局

- 桌面宽画布使用 4 列非对称拼贴，保证主卡片和次级卡片之间的层级关系。
- 1:1 和 4:5 画布收紧间距并切换为更窄的 2 列布局，避免内容被舞台裁切。
- 9:16 画布使用单列或双列紧凑布局，保持所有卡片可见，聚焦卡片仍以舞台中心为目标。
- 布局参数以容器尺寸为基准，不依赖固定视口像素。

## 实现边界

- 修改 `app.js` 中 Mosaic 的生成、聚焦状态和编辑态事件处理。
- 修改 `styles.css` 和 `index.html` 中对应的 Mosaic 内嵌/外部样式，保持两份样式同步。
- 同步发布态内嵌脚本中的聚焦和退出逻辑。
- 不修改 Orbit Carousel、Circular Gallery、Card Totem 的布局与交互。

## 验证标准

- 编辑预览和发布预览均能点击聚焦、Esc 退出、点击空白退出。
- 1:1、4:5、9:16、16:9 下卡片不被舞台边界裁切，整体占比明显高于当前版本。
- 聚焦和恢复使用平滑过渡，没有突然跳动或页面滚动。
- `node --check app.js` 和 `git diff --check` 通过。
